COMMAND 1
---------

Command

lsblk -o NAME,SIZE,TYPE,FSTYPE,MOUNTPOINT

Purpose

Displays all block storage devices connected to the system.

Output

The command listed available disks, partitions, filesystem types, and
mount points.

Observation

The output provided a complete overview of the storage architecture,
allowing identification of all available storage devices and their
associated mount locations.

---------------------------------------------------------------------------

COMMAND 2
---------

Command

df -hT

Purpose

Displays mounted file systems with filesystem type and human-readable
disk usage statistics.

Output

Displayed filesystem names, total capacity, used space, available space,
usage percentage, and mount points.

Observation

The mounted filesystems were functioning correctly. Disk utilization
remained well below critical levels, indicating sufficient storage
capacity for current workloads.

---------------------------------------------------------------------------

COMMAND 3
---------

Command

df -i

Purpose

Displays inode utilization for each mounted filesystem.

Output

Displayed total, used, and available inodes together with utilization
percentages.

Observation

The filesystem had a large number of free inodes available. This confirms
that additional files can be created without encountering inode
exhaustion.

---------------------------------------------------------------------------

COMMAND 4
---------

Command

df -h --total

Purpose

Displays the combined disk usage across all mounted filesystems.

Output

Displayed total storage capacity, used space, available space, and
overall utilization percentage.

Observation

The combined storage utilization remained within healthy operating
limits, demonstrating efficient storage management across the system.
